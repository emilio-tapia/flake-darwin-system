# Changelog — v1.0.0

**Fecha:** 2026-02-21
**Tipo:** feat
**Alcance:** general

## Descripción

Carga inicial del script de seguimiento de requerimientos (REQ monitor) como proyecto Google Apps Script.

## Cambios realizados

1. Se creó la estructura base del proyecto con archivos de configuración (.clasp.json, appsscript.json, package.json)
2. Se implementó el sistema de registro de requerimientos vía Gmail con detección de asunto `[REQ]`
3. Se desarrolló el motor de estados (Status.gs) para gestionar el ciclo de vida de cada requerimiento
4. Se crearon utilidades compartidas (Utils.gs) para funciones auxiliares reutilizables
5. Se implementó el sistema de caché (Cache.gs) para optimizar lecturas de hojas de cálculo
6. Se añadió el sistema de alertas (Alerts.gs) para notificaciones por correo
7. Se creó el módulo de logs (Logs.gs) para diagnóstico y auditoría
8. Se implementó la configuración de empleados (Config_Employees.gs) y diseño de columnas (Layout_Sheets.gs)
9. Se desarrolló el setup inicial de hojas (Setup.gs) para crear pestañas por empleado
10. Se redactó el manual de operaciones (MANUAL.txt)

## Archivos modificados

- `.clasp.json` — configuración de clasp para despliegue
- `Config.gs` — constantes del monitor REQ (prefijo, intervalos, IDs)
- `Config_Employees.gs` — mapa de empleados con IDs, nombres y correos
- `Layout_Sheets.gs` — definición de columnas y anchos para pestañas REQ
- `Main.gs` — punto de entrada `runReqTracker()` con ciclo principal
- `Register.gs` — registro de nuevos requerimientos desde correos entrantes
- `Status.gs` — procesamiento de respuestas y transiciones de estado
- `Alerts.gs` — envío de alertas y recordatorios por correo
- `Cache.gs` — sistema de caché para datos de hojas de cálculo
- `Logs.gs` — funciones de logging y diagnóstico
- `Utils.gs` — utilidades generales (parseo de correos, formateo)
- `Setup.gs` — inicialización de hojas por empleado
- `Code.gs` — funciones de entrada expuestas al entorno GAS
- `MANUAL.txt` — documentación de despliegue y operación
- `appsscript.json` — manifest del proyecto GAS

## Impacto para el usuario

Se establece el sistema base de seguimiento de requerimientos: los supervisores pueden enviar correos con asunto `[REQ]` y el sistema registra automáticamente las tareas en hojas de cálculo individuales por empleado, con seguimiento de estados.
