# Changelog — v1.2.0

**Fecha:** 2026-05-14
**Tipo:** feat
**Alcance:** darwin

## Descripción

Nuevo módulo darwin `customPackages` para empaquetar herramientas Python/Go/npm directamente desde GitHub cuando no están en nixpkgs ni Homebrew. Migración de pnpm a Homebrew para acceder a la serie 11.x. Activación de `autoUpdate` en Homebrew. Desactivación de vscode.

## Cambios realizados

1. Se añadió `darwinModules/development/customPackages.nix` con helper `mkPythonAppFromGitHub` y plantillas comentadas para `buildGoModule` y `buildNpmPackage`. Incluye `dirsearch` como primer paquete; deja `dhv` documentado pero bloqueado (falta `textual-fspicker` en nixpkgs 25.05).
2. Se registró el módulo en `flake.nix` y se activó `customPackages.enable = true` en m4Pro y macbookPro.
3. Se removió `pnpm` de `environment.systemPackages` en `devTools.nix` y se descomentó `"pnpm"` en los `brews` de ambos hosts. Motivo: `nixpkgs-25.05-darwin` sólo ofrece la serie 10.x; Homebrew tiene la 11.x.
4. Se activó `homebrew.onActivation.autoUpdate = true` en ambos `homeBrew.nix`.
5. Se comentó `vscode` en `darwinModules/desktopApps.nix` (en desuso).

## Archivos modificados

- `darwinModules/development/customPackages.nix` — módulo nuevo (helper + plantillas + dirsearch)
- `flake.nix` — import del nuevo módulo en `mkDarwinSystem`
- `hosts/m4Pro/configuration.nix` — habilita `customPackages.enable`
- `hosts/macbookPro/configuration.nix` — habilita `customPackages.enable`
- `darwinModules/development/devTools.nix` — `pnpm` comentado, gestionado por Homebrew
- `hosts/m4Pro/homeBrew.nix` — `"pnpm"` añadido a `brews`, `autoUpdate = true`
- `hosts/macbookPro/homeBrew.nix` — `"pnpm"` añadido a `brews`, `autoUpdate = true`
- `darwinModules/desktopApps.nix` — `vscode` comentado

## Impacto para el usuario

- `pnpm --version` debería empezar por `11.` tras el siguiente `darwin-rebuild switch` y abrir una shell nueva. El binario pasa de `/run/current-system/sw/bin/pnpm` (nix) a `/opt/homebrew/bin/pnpm` (M4) o `/usr/local/bin/pnpm` (Intel).
- `dirsearch` disponible system-wide vía nix.
- Homebrew se actualiza automáticamente al activar la configuración.
- VS Code deja de instalarse; si vuelve a hacer falta, descomentar la línea en `desktopApps.nix`.
