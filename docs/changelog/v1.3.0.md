# Changelog — v1.3.0

**Fecha:** 2026-06-02
**Tipo:** feat
**Alcance:** home-manager

## Descripción

Unificación y mejora del stack de terminal (Alacritty + WezTerm + tmux) en ambos hosts. WezTerm pasa a gestionarse de forma declarativa en Nix (replicando la fuente y la paleta oceánica de Alacritty), tmux arranca automáticamente al abrir Alacritty o WezTerm, y se corrigen inconsistencias de configuración de tmux junto con el soporte de true color para WezTerm.

## Cambios realizados

1. Se añadió `programs.wezterm` en `home_manager/emilio_m4Pro.nix` y `home_manager/admin_macbookPro.nix`, configurado en Lua vía `extraConfig`: fuente `MesloLGS NF` (fallback `FiraCode Nerd Font Mono`), tamaño 13, opacidad 0.95, padding, cursor bloque parpadeante, scrollback 100000, shell `${pkgs.zsh}/bin/zsh` y la paleta oceánica espejo de Alacritty.
2. Se migró WezTerm de cask de Homebrew a Nix: se comentó `"wezterm"` en `hosts/m4Pro/homeBrew.nix`.
3. Se añadió el auto-arranque de tmux en `programs.zsh.initContent` (usando `lib.mkBefore` para correr antes del resto del init): si la shell es interactiva, no estás ya dentro de tmux (`$TMUX` vacío) y el terminal es Alacritty (`$ALACRITTY_WINDOW_ID`) o WezTerm (`$TERM_PROGRAM`), hace `exec tmux new-session` (sesión nueva e independiente por ventana).
4. Se corrigió `programs.tmux.terminal` de `screen-256color` a `tmux-256color` en ambos hosts, eliminando la línea `set -g default-terminal` redundante del `extraConfig`.
5. Se añadió el override de true color de tmux para WezTerm (`set -ga terminal-overrides ",xterm-wezterm:RGB"` y `:Tc`) en ambos hosts.
6. Se eliminó el `tmux` duplicado de `home.packages` en emilio (ya lo provee `programs.tmux`).
7. En macbookPro, el `terminal.shell` de Alacritty volvió a ser un login shell de zsh, sustituyendo el hack previo `zsh -l -c "tmux new -A -s main"` (sesión compartida) por el nuevo arranque desde el `.zshrc`.

## Archivos modificados

- `home_manager/emilio_m4Pro.nix` — bloque `programs.wezterm` nuevo; auto-arranque de tmux en `initContent`; tmux `terminal = "tmux-256color"` + override wezterm; dedupe de tmux en `home.packages`
- `home_manager/admin_macbookPro.nix` — bloque `programs.wezterm` nuevo; auto-arranque de tmux en `initContent`; tmux `terminal = "tmux-256color"` + override wezterm; Alacritty `terminal.shell` vuelve a zsh plano
- `hosts/m4Pro/homeBrew.nix` — cask `"wezterm"` comentado (gestionado por Nix)

## Impacto para el usuario

- Al abrir una ventana nueva de Alacritty o WezTerm, se entra directamente a una sesión de tmux propia (no afecta a Terminal.app, la terminal de VS Code ni sesiones SSH).
- WezTerm se instala vía Nix; en m4Pro el cask de Homebrew se retira en el próximo `darwin-rebuild switch`. El binario de Nix incluye su propio `WezTerm.app`.
- Colores de 24-bit (true color) garantizados dentro de tmux tanto en Alacritty como en WezTerm.
- macbookPro pasa de una sesión tmux compartida (`main`) a una sesión independiente por ventana.
